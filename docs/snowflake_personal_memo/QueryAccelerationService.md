Applied at the Warehouse level (Data science or BI recommanded)

Scan faster

Trigger: Snowflake automatically detects if a query will scan massive data and calls the QAS pool to handle the scan phase (potentially saving minutes/hours of processing).

Scope Limit: Only accelerates Scans & Filters. Does not help with complex Joins or UDFs.

Serverless feature.

Can avoid you to scale up your WH

Cost : Can be found in SNOWFLAKE.ACCOUNT_USAGE.QUERY_ACCELERATION_HISTORY + WH name which used it
